# dictate — user dictionary for whisper initial_prompt (lexical bias)
#
# Copy this to ~/.config/whisper/dictionary.txt and edit. The daemon reads it once at
# startup and seeds whisper's decoder with these words, raising recognition of YOUR
# vocabulary without changing the model. Re-read requires a daemon restart:
#     launchctl kickstart -k gui/$(id -u)/com.user.dictate
#
# WHAT TO PUT HERE (highest value first — a tight, targeted list beats a long generic one):
#   1. ACTION VERBS / workflow words you say all day inside Russian — their English roots
#      get transliterated wrong ("закоммить", "отрефактори", "задеплой"), so biasing toward
#      the English form fixes them. This is usually the biggest win.
#   2. PERSONAL infra / tooling proper-nouns you dictate constantly — your servers, your
#      stack, the tools you live in (vps, ssh, backend, your CLI/service names…). These are
#      high value: whisper has never seen them and mangles them every time. (Generic, widely
#      known framework names you rarely misdictate are NOT worth a slot — see below.)
#   3. English-in-Russian jargon you use a lot (prompt, endpoint, label, toast…).
# What NOT to bother with: common framework/tool names whisper already nails, generic words
# it gets right, and per-project UI churn (this week's feature nouns). A term only earns a
# slot if whisper actually gets it wrong AND you say it across projects, not just today.
#
# HOW TO BUILD THIS FROM REAL DATA (recommended): mine your own dictation history instead of
# guessing. If you dictate mostly into Claude Code, your past takes live as Russian user
# messages in ~/.claude/projects/**/*.jsonl. Extract the Russian-dominant messages, strip
# code/paths/camelCase, and rank the English tokens that sit *inside* Russian sentences —
# those are exactly the words whisper transliterates wrong. Frequency = priority.
#
# Format:
#   • one word or short phrase per line; ORDER = PRIORITY (most important first — if the
#     budget overflows, the earliest lines win and the rest are dropped).
#   • '#' starts a comment to end of line, but only at line start or after a space, so a
#     term like  C#  or  F#  is kept verbatim.
#   • blank lines are ignored; exact duplicates are dropped (first one kept).
#
# Budget is ZERO-SUM: the prompt is capped at ~224 tokens (a long/prose-like prompt makes the
# turbo model loop — project gotcha #11/#21), so keep it a SHORT comma-list of words, not
# prose. ~50 short words fills it. Adding a term means cutting a weaker one. The per-entry
# estimate (src/dictate_dict.h) leans HIGH on purpose, so the real whisper token count sits
# well under 224 — watch the daemon's logged "~N tok" and keep it near 224, not far above.
# Multi-word phrases ("pull request") are costly; prefer the short form ("PR"). Raise the cap
# via WHISPER_PROMPT_MAXTOK (in the LaunchAgent plist's EnvironmentVariables) if you must.
# Toggle the whole feature off with WHISPER_PROMPT=0; override the path with WHISPER_DICT.
#
# Below is an ILLUSTRATIVE template — replace with the words YOU actually say.
